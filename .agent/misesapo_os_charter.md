# ミセサポOS：ユニバーサル・マスター・ブリーフ (UMB)
## 統合システム定義・思考プロセス共有ドキュメント

📅 **最終更新**: 2026-01-13
🎯 **ステータス**: プロジェクトマスター定義

## 1. プロジェクト・ビジョン
店舗経営の「不確実性（清掃・修繕・管理のムラ）」を、AIと金融ガバナンスによって「確実な資産管理」へと昇華させる店舗用次世代OS。

## 2. ミセサポ憲章 (The Core Logic)
全ての挙動、判定、精算はこの3条に基づき自律的に実行される。

1. **無謬性の追求**: 主観を廃し、全ての価値をデジタルな「情報」として定義する。
2. **結果の成立義務**: プロセスではなく、状態の改善（Result）に対してのみ対価が発生する。
3. **情報の信頼性**: AIアシスタント『MISOGI』による客観的検収により、人的ミスを構造的に排除する。

## 3. 金融・管理モデル (The Fintech Engine)
- **通貨体系**: 先払い500ポイント信託モデル（1P = 1,000円換算）。
- **ガバナンス（180日サイクル）**: 資金決済法準拠のため有効期限を6ヶ月に設定。実運用はこの6ヶ月を1サイクルとし、契約の更新・切り替えを行う。
- **ポイント転換**: 失効直前のポイントは強制的に「ブランディングメニュー」等に転換。
- **ミセサポTrust**: 店舗の管理実績（スコア）に応じた「緊急与信枠」の付与。
- **ミセサポ共済**: プラットフォーム全体で包括賠償保険を組成。

## 4. 24ヶ月の物語 (The Narrative)
店舗LTVを最大化するための4フェーズ構成（各6ヶ月＝1サイクル）。

- **Q1: 破壊と再生 (1-6M)**: 重度汚染（Cランク）の徹底排除。最初の180日。
- **Q2: 安定と認証 (7-12M)**: Aランク維持。ミシュラン一つ星認定。
- **Q3: 最適化と効率 (13-18M)**: 予防管理への移行。メンテナンスコストの低減。
- **Q4: ブランディング (19-24M)**: 浮いたコストを集客・PRへ転換。3つ星認定。
3つ星認定。

## 5. AI『MISOGI』解析プロトコル
Gemini Vision APIを活用した画像解析の重み付け定義。

### 減点係数 (Weights)
| 項目 | 係数 | 対象・リスク |
|:---|:---|:---|
| **油汚れ (Grease)** | 5.0 | 火災・閉塞リスク |
| **生物汚れ (Bio)** | 4.0 | 食中毒・カビ |
| **水垢 (Scale)** | 3.0 | 物理劣化 |
| **不備 (Damage)** | 2.0 | 管理不足 |
| **埃 (Dust)** | 1.0 | 美観 |

### 判定ランク
- **A (80-100)**: 維持。清潔資産。ミシュラン認定。
- **B (40-79)**: 是正。日常清掃の限界。
- **C (0-39)**: 介入。資産価値毀損（火急の対応が必要）。

## 6. 技術スタック (Technical Architecture)
- **Frontend**: Cyberpunk/Neon UI (Vanilla JS/CSS).
- **Backend**: AWS Lambda (Python), API Gateway.
- **Database**: DynamoDB (Workers, Stores, Schedules, Media).
- **Storage**: Amazon S3 (Lifecycle Policy による24ヶ月メディア保存).
- **AI**: Google Gemini API (Vision / Text / Web Extraction).

## 7. UIデザイン指針
- **UIコンセプト**: 「サイバーパンク / ネオン / CRTエフェクト」。
- **連動計器**: UI上の全てのメーターやネオンカラーの変化は、上記「AI解析プロトコル」の数値と1対1で連動させる。
